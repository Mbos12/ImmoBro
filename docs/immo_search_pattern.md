# Standard Immo Search Pattern

Canonical criteria:

```json
{
  "transaction": "for_sale",
  "propertyTypes": ["house", "apartment"],
  "maxPriceEur": 300000,
  "minLivingAreaM2": 50,
  "epcLabels": ["A++", "A+", "A", "B", "C"],
  "includePriceOnRequest": false,
  "includeUnknownSurface": false,
  "sort": "newest"
}
```

Interpretation:

- EPC C minimum means A++, A+, A, B, or C. Do not include D.
- Skip listings without a numeric price.
- Skip listings below 50 m2, and skip listings where the website cannot apply or expose the surface filter.
- Search houses and apartments. Some sites require separate URLs for each property type.

## Immoscoop

Confirmed from URL structure:

```text
https://www.immoscoop.be/zoeken/te-koop/{city}/{propertyType}?maxPrice=300000&minLivableSurfaceArea=50&epcLabels=A%2B%2B%2CA%2B%2CA&epcLabels=B&epcLabels=C
```

Use one URL per property type:

```text
https://www.immoscoop.be/zoeken/te-koop/genk/appartement?maxPrice=300000&minLivableSurfaceArea=50&epcLabels=A%2B%2B%2CA%2B%2CA&epcLabels=B&epcLabels=C
https://www.immoscoop.be/zoeken/te-koop/genk/woonhuis?maxPrice=300000&minLivableSurfaceArea=50&epcLabels=A%2B%2B%2CA%2B%2CA&epcLabels=B&epcLabels=C
https://www.immoscoop.be/zoeken/te-koop/hasselt/appartement?maxPrice=300000&minLivableSurfaceArea=50&epcLabels=A%2B%2B%2CA%2B%2CA&epcLabels=B&epcLabels=C
```

Notes:

- The user-provided example included `epcLabels=D`; remove it for EPC C minimum.
- `appartement` and `woonhuis` are the observed type path segments.

## Zimmo

Confirmed base approach: Zimmo packs filters into a base64-encoded JSON `search` parameter.

Canonical filter object:

```json
{
  "filter": {
    "status": { "in": ["FOR_SALE", "TAKE_OVER"] },
    "placeId": { "in": ["{PLACE_ID}"] },
    "price": { "range": { "max": 300000 }, "unknown": false },
    "floorspaceSurface": { "range": { "min": 50 }, "unknown": false },
    "category": { "in": ["HOUSE", "APARTMENT"] },
    "epcLabel": { "in": ["A++", "A+", "A", "B", "C"] }
  },
  "paging": { "from": 0, "size": 20 },
  "sorting": [{ "type": "DATE", "order": "DESC" }]
}
```

URL pattern:

```text
https://www.zimmo.be/nl/zoeken/?search={BASE64_URL_ENCODED_JSON}&p=1#gallery
```

Notes:

- `placeId` must be discovered per city from a Zimmo city page. Genk was observed as `2460`; Hasselt was observed as `2475`.
- `price.unknown=false` is the important part that skips price-on-request listings.
- The exact EPC key should be validated against a Zimmo URL generated by its own UI before relying on automation. The canonical field above is the normalized adapter field.
- Hasselt path-based test URL shape:

```text
https://www.zimmo.be/nl/hasselt-3500/te-koop/appartement/?search={BASE64_URL_ENCODED_JSON_WITH_PLACEID_2475_OR_PATH_LOCATION}&p=1#gallery
```

For Hasselt, the rendered page still showed some broader city results, so the dashboard import applied the canonical numeric checks again on the visible cards before adding rows.

## Immoweb

Confirmed from URL structure:

```text
https://www.immoweb.be/nl/zoeken/huis-en-appartement/te-koop/{city}/{postalCodeOrRegion}?countries=BE&maxPrice=300000&page=1&orderBy=newest
```

Observed Genk base:

```text
https://www.immoweb.be/nl/zoeken/huis-en-appartement/te-koop/genk/3600?countries=BE&maxPrice=300000&page=1&orderBy=newest
https://www.immoweb.be/nl/zoeken/appartement/te-koop/hasselt/3500?countries=BE&maxPrice=300000&page=1&orderBy=newest
```

Adapter fields to add/validate from Immoweb's advanced-search UI:

```json
{
  "minLivingAreaM2": 50,
  "epcLabels": ["A++", "A+", "A", "B", "C"],
  "includePriceOnRequest": false
}
```

Notes:

- Immoweb reliably accepted `maxPrice`, location, property family, and newest sorting in the observed URLs.
- Surface/EPC/unknown-price parameter names should be generated once via Immoweb's UI and then captured, because guessing those query keys risks silently wrong results.
